# 課題台帳（Challenge Ledger）

> このエージェントが扱う課題の台帳（正本）。人間は**共有ソース**に課題を書き、このエージェントが**自分に関係する分だけ**ここへ取り込む（ingestion）。
> 記入形式は**下の記入例**のとおり（人間記入欄＋分類欄）。それだけで運用できる。
> （このファイルは claude-flywheel プラグインの雛形から生成）
>
> - **人間（共有ソース）**: 「人間記入欄」に困りごと・期待を書く（ドメインやサービスを知らなくてよい）。
> - **エージェント**: 取り込み時に「分類欄」を埋める（自分のどのポジションが引き受けるか）。

## 記入例（コピーして使う）

```markdown
### [C-001] <タイトル>

**人間記入欄**
- 起票者 / 起票日: <name> / <YYYY-MM-DD>
- 説明: <背景・困っていること・期待する状態>
- 完了条件（任意）: <こうなれば完了>
- 体感の緊急度（任意）: 高 | 中 | 低

**分類欄（エージェントが記入）**
- 担当ポジション:
- 関連サービス:
- 優先度: P0 | P1 | P2
- ステータス: 未分類（未分類 → 分類済 → 計画承認待ち → 着手中 → 検証中 → 完了確認待ち → 完了）
- タスク案: （run-cycle の計画ステップが記入）
- 取り込み元: （外部ソースから取り込んだ場合のみ ingest-challenges が付与。手書き課題は空でよい）
- 備考:
```

> **人間の承認方法**: `計画承認待ち → 着手中`（タスク案の承認）と `完了確認待ち → 完了`（完了の承認）の 2 遷移だけは、人間がステータス行を直接編集して進める（編集そのものが承認）。差し戻しはステータスを戻す。
