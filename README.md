# claude-flywheel

AI を「自律的に働く社員」として動かすための **ハーネス・ツール・運用ナレッジ** を管理するリポジトリです。

指定された目標に対して、AI 自身がやるべきことを探索し、タスクに分解し、実行・検証・改善まで自走させる仕組みを継続的に磨き込んでいきます。開発業務・運営業務の両方を対象とします。

**最重要の目標は、AI 自身が課題を整理し、自分のポジション（役割）に関係するものを選び取って自律的に動くエージェントを作ること** です。人間が一つずつ指示するのではなく、エージェントが「自分が取り組むべき課題」を判断して動き出す状態を目指します。

そのため本プロジェクトの主目的は、特定の業務やツールに縛られない **自走するエージェントのための汎用的な環境（プラットフォーム）** を作ることです。具体的には ①ポジションに沿ったスキル群、②エージェントを自律的に動かすアプリケーション／実行基盤 を提供します。開発の自律化では [claude-harness](https://github.com/masanami/claude-harness) を**利用できます**が、本プロジェクトは特定ツールに**依存しません**（差し替え可能）。

## コンセプト：Flywheel（弾み車）

目標設定 → タスク探索 → 実行 → 検証 → 学習（記憶・改善）のループを回し続け、回せば回すほど自律性と成果が加速していく状態を目指します。

```
        ┌─────────────┐
        │  目標設定    │
        └──────┬──────┘
               ▼
   ┌──────────────────────┐
   │   タスク探索・分解     │  ← AI が「やるべきこと」を発見
   └──────────┬───────────┘
              ▼
        ┌──────────┐
        │   実行    │
        └────┬─────┘
             ▼
        ┌──────────┐
        │   検証    │
        └────┬─────┘
             ▼
   ┌──────────────────────┐
   │  学習（記憶・改善）     │  ──┐
   └──────────────────────┘    │
              ▲                 │
              └─────────────────┘  弾み車を回し続ける
```

## 配布形態：Claude Code プラグイン

claude-flywheel は **Claude Code プラグイン**として install して使います。**機械（スキル・テンプレ）はプラグイン**、**状態（課題台帳・positions・memory）は利用先ワークスペース**に置く、という分離が前提です（プラグインは配布・更新されるため、運用状態は中に持ちません）。

```
# 1. マーケットプレイスを登録し、プラグインを install
/plugin marketplace add masanami/claude-flywheel
/plugin install claude-flywheel@claude-flywheel

# 2. 利用先ワークスペースで状態を初期化
/flywheel-init                # 状態を生成（templates から）
/bootstrap-domain-map         # positions/・memory/ を生成（ドメイン地図）

# 3. 課題を記入して自走
#   challenge-ledger.md に課題を記入 → /run-cycle（routine で定期実行も可）
```

提供物（プラグイン本体）:
- **`skills/`** — `flywheel-init` / `bootstrap-domain-map` / `run-cycle` などのスキル群【成果物(a)】
- **`templates/`** — 利用先に scaffold する雛形（課題台帳・ポジション・ランタイム設定）
- **`docs/`** — 設計ドキュメント（要件・アーキテクチャ・課題・memory運用）

> 同じプラグインを複数のワークスペースに導入でき、それぞれが独立した状態を持ちます（汎用性）。設計の詳細は [docs/architecture.md](docs/architecture.md) を参照。
